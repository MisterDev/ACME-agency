Rendere un business process eseguibile:

 - In Eclipse creare un nuovo progetto Maven
 - Settare group e artifact id e nome, e packaging war
 - Aggiungere le dipendenze di Camunda nel file pom.xml (esempio nel file pom.xml)
 - (Testare il corretto funzionamento effettuando un Run > maven install)
 - Creare un nuovo package in src/main/java
 - Creare nel package un classe di tipo Process Application (esempio nel file SumProcessApplication.java)
 - Aggiungere in src/main/resources/META-INF (eventualmente creare META-INF) il file processes.xml (esempio nel file processes.xml)

 - Da Camunda Modeler creare un modello di processo
 - Settare il processo Executable e specificare un Id
 - Configurare eventuali User Task specificando l'Assignee
 - Configurare eventuali Service Task specificando le classi Java da eseguire e
   per ogni task eseguibile creare una classe che faccia da delegate (esempio PerformSumDelegate.java)

 - E' possibile specificare quali parametri si aspettano le varie fasi del processo con file html
 - Esempio di form in /src/main/webapps/forms/sumrequest.html
 - In Camunda si specifica quindi embedded:app:forms/sumrequest.html nel tab dei form del corrispondente task

 - Salvare il bpmn in src/main/resources

 - Run > maven install e mettere il file .war ottenuto in server/wildfly-10.0.0.Final/standalone/deployments di Camunda Wildfly


Per piu dettagli: https://docs.camunda.org/get-started/bpmn20/project-setup/
