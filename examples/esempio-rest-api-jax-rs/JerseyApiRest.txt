Semplice test con Jersey e Jax-RS, per un servizio che usi Rest sempre all'interno di Wildfly

 - Nuovo progetto Maven, specificando packaging war, poi aggiustare il pom.xml con le dipendenze di Jeresy (vedi pom.xml)
 - Creare un package in cui inserire una classe che estenda Application (vedi RestApp.java), in 
   cui specificare il path dell'applicazione 
 - In un diverso package, creare una classe per ogni servizio, in cui all'interno specificare i comportamenti
   per le possibili richieste HTTP (vedi FirstService.java e SecondService.java)
 - Run as > maven install e fare il deploy del .war in Camunda Wildfly

 - A questo punto e' possibile fare le richieste all'indirizzo che e' dato dalla composizione dei path scelti nelle classi Java
   (ex: http://localhost:8080/rest-api-test-0.1/rest/test/ex )
